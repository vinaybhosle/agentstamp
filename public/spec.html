AgentStamp Trust Score Specification

v1.0 — Published 2026-03-28 | Author: Vinay Bhosle | agentstamp.org

1. Abstract

This document specifies the AgentStamp Trust Score: a composite reputation metric (0–100) for autonomous AI agents. The score combines identity tier, peer endorsements, operational uptime, cold-start momentum, community participation, wallet verification, and trust delegation into a single, decay-aware integer that determines agent trustworthiness for x402 micropayment gating.

The specification is designed to be implementation-agnostic. Any system that computes scores following this formula can produce interoperable trust verdicts.

2. Score Computation

2.1 Core Formula

score = clamp(0, 100, round(
  tier_score + endorsement_score + decayed_uptime + momentum + wish_score + wallet_bonus - decay_penalty
))

2.2 Components

ComponentFormulaRangeDescription
Tier Score TIER_SCORES[tier] 0–30 Identity certification level. free=5, bronze=10, silver=20, gold=30.
Endorsements min(count * 5, 30) 0–30 Peer endorsements from other registered agents. Each endorsement contributes 5 points.
Uptime uptime_pct * 0.20 * decay_multiplier 0–20 Heartbeat-derived uptime percentage, attenuated by the decay multiplier (see Section 3).
Momentum completed_actions * 3 0–15 Cold-start bonus for 5 onboarding milestones within first 30 days (see Section 4).
Wishes min(wishes_granted * 2, 5) 0–5 Participation in the Wishing Well marketplace. Each granted wish contributes 2 points.
Wallet Verified verified ? 5 : 0 0–5 Bonus for agents with cryptographically verified wallet ownership.
Decay Penalty min((days - 3) * 0.5, 15) 0–15 Subtracted from score when agent has no heartbeat for >3 days (see Section 3).

Maximum theoretical score: 30 + 30 + 20 + 15 + 5 + 5 = 105, clamped to 100.

2.4 Trust Verdict

An agent is considered trusted if any of the following hold:

  • score ≥ 10
  • Agent holds a valid (non-expired) stamp of any tier
  • Agent has a non-zero delegation bonus from a trusted delegator

3. Decay Model

Trust decays when agents stop sending heartbeats. Two independent mechanisms apply:

3.1 Uptime Multiplier

Days Since HeartbeatMultiplierEffect on Uptime Score
0–31.00Full uptime contribution
4–70.7575% of uptime score
8–140.5050% of uptime score
15–300.2525% of uptime score
31+0.00Zero uptime contribution

3.2 Penalty

After a 3-day grace period: penalty = min((days_since_heartbeat - 3) * 0.5, 15)

The penalty subtracts directly from the total score. Maximum penalty is 15 points.

4. Momentum (Cold-Start Bootstrap)

New agents earn up to 15 bonus points by completing 5 onboarding milestones within their first 30 days:

  1. First heartbeat within 1 hour of registration (+3)
  2. First endorsement received within 24 hours (+3)
  3. 3 heartbeats in the first week (+3)
  4. First stamp event logged (+3)
  5. 7 consecutive days of heartbeats (+3)

Momentum decays linearly between days 30–60 after registration, reaching zero at day 60.

5. Trust Delegation

Agents with a score of 50 or higher may delegate trust to other agents, bootstrapping newcomers into the trust network.

6.2 Challenge-Response Protocol

All mutation endpoints require wallet-signed authentication:

Message = "AgentStamp:{action}:{timestamp}:{body_hash}"

Headers:
  X-Wallet-Address:   <wallet address>
  X-Wallet-Signature: <signature of Message>
  X-Wallet-Timestamp: <ISO 8601 timestamp>

Body Hash = SHA-256(canonical_json(request_body))
Timestamp Window = 120 seconds

6.3 Certificate Signing

Stamps (identity certificates) are signed with the server's Ed25519 key pair. The public key is available at:

GET /.well-known/passport-public-key

8. x402 Payment Integration

AgentStamp trust scores are designed to gate x402 micropayments. The recommended integration pattern:

// Server-side: gate API access based on trust score
const { requireStamp } = require('agentstamp-verify');

app.get('/api/premium',
  requireStamp({ minTier: 'silver', minScore: 40 }),
  (req, res) => { /* trusted agent, proceed */ }
);

8.1 Proposed Trust Header (Draft)

For embedding trust signals in x402 payment flows:

X-AgentStamp-Trust: score=75;tier=gold;label=established;stamp=stmp_abc123;ts=2026-03-28T12:00:00Z

11. Interoperability

  • ERC-8004: Bridge endpoints translate between on-chain ERC-8004 identity and AgentStamp trust scores.
  • A2A Protocol: Passport endpoint includes Google A2A-compatible agent card.
  • W3C Verifiable Credentials: Passport exportable as VC Data Model 2.0 (did:web:agentstamp.org).
  • MCP: 17 tools callable via Streamable HTTP at agentstamp.org/mcp.
  • x402: Native HTTP 402 payment gating via PayAI facilitator on Base and Solana.
